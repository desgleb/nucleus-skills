# Daily Digest Skill

Дневная сверка: почта (дельта) + план недели (Wiki) + задачи (Трекер) → GTD-фильтр "что делать прямо сейчас".

## Параметры

- Аргумент: нет.

## Константы

- **Пользователь Wiki**: `users/g.deshin`
- **Lead (ответственный)**: `g.deshin`
- **Slug страницы недели**: `users/g.deshin/plany-rabot-po-nedeljam/YYYY-WNN`
- **State-файл**: `/Мерусофт/state/skill-state.json` (Яндекс Диск)
- **Почтовые папки**: `["INBOX", "Outbox", "Sent", "tracker"]`

## State-файл

Общий с `good-morning` и `happy-evening` — файл на Яндекс Диске.

**Чтение** (начало навыка):
```
download_disk_file(path="/Мерусофт/state/skill-state.json")
```
Если файл не найден — считать все timestamps = `null`.

**Запись** (конец навыка):
```
upload_to_disk(path="/Мерусофт/state/skill-state.json", content=<json>, overwrite=true)
```
Обновить `last_mail_read` и `last_digest`, сохранить остальные ключи без изменений.

## Отличие от good-morning

| | good-morning | daily-digest |
|---|---|---|
| Когда | Утро, 1 раз в день | Любое время, сколько угодно раз |
| Почта | Полная с last_mail_read | Дельта с last_mail_read |
| План недели | Создаёт / актуализирует | Только читает, не меняет |
| Wiki проектов | Обновляет | Не трогает |
| Задачи Трекер | Глубокое чтение по приоритетам | Быстрая проверка статусов задач из плана |

| Публикация | Wiki | Только в чат |

## Архитектура

**Три источника → один вывод.**

```
Почта (дельта)  ──┐
                   ├──→  GTD-фильтр  ──→  "Что делать прямо сейчас"
План недели (Wiki) ┤
                   │
Трекер (статусы)  ─┘
```

**Принцип:** навык НЕ меняет данные (не пишет в Wiki, не обновляет план). Только читает, анализирует и рекомендует. Изменения — через `/good-morning` или `/happy-evening`.

## Алгоритм

### Шаг 1 — Подготовка

1. Скачать state с Диска: `download_disk_file(path="/Мерусофт/state/skill-state.json")` → взять `last_mail_read`
2. Определить текущую ISO-неделю, slug плана

### Шаг 2 — Сбор данных (3 источника, последовательно)

#### 2a. Почта (дельта)

```
get_mail_summary(since=<last_mail_read>, before=<завтра>, folders=["INBOX","Outbox","Sent","tracker"])
```

Анализ:
- Новые входящие → кто, о чём, нужен ли ответ?
- Исходящие → что отправил, на что ответил
- Tracker → @упоминания, изменения задач
- Открытые цепочки (входящее без исходящего) → **ждут ответа**

Если важные письма — прочитать содержимое через `get_mail(uid)`.

> **⛔ Гейт «мяч у нас» (ОБЯЗАТЕЛЬНО перед пометкой «Делай сейчас / нужен ответ»):** `get_mail_summary` отдаёт только ЗАГОЛОВКИ. По каждой открытой цепочке найти САМОЕ ПОЗДНЕЕ письмо по теме (входящее ИЛИ исходящее) и открыть его тело через `get_mail(uid)` — не судить по заголовку:
> - последнее письмо в треде — наш **Sent** (ответ / вложения / оценка) → цепочка **закрыта / в Ожидании**, мяч у клиента, НЕ «Делай сейчас»;
> - последнее — **входящее без нашего ответа** → действительно ждут нас.
>
> Проверять и исходящие РП (Храмов `s.khramov`, Растворцев) — задачу мог закрыть он, а не мы. Пропуск Sent = ложное «срочно» и подрыв доверия к навыку.

#### 2b. План недели (Wiki)

```
get_wiki_page(slug=<текущая_неделя>)
```

Извлечь ВСЁ:
- **Q1**: задачи, статусы `[ ]` / `[x]`, контекст
- **Q2**: задачи, статусы
- **Q3**: задачи, статусы
- **Q4**: задачи, статусы
- **Лягушка дня**: какая на сегодня? Сделана?
- **Слон дня**: какой кусочек на сегодня?
- **Почтовый контекст**: что ожидалось?

#### 2c. Задачи Трекер (выборочно)

По каждой **незакрытой** задаче из плана (Q1-Q4), у которой есть ключ Трекера (MIMP-xxx, MDEV-xxx):

```
get_issue(issue_key)
```

Проверить:
- Статус изменился? (closed/resolved → отметить)
- Новые комментарии с момента последней проверки?
- Дедлайн приближается/просрочен?

**Rate limit:** не более 5-7 запросов get_issue. Приоритет: Q1 → Q2 → лягушка/слон. Q3/Q4 — только если есть tracker-нотификации.

Если в комментариях есть прямой запрос — прочитать через `manage_comments(operation="list", target={type:"issue", id:issue_key})`.

### Шаг 3 — GTD-классификация

Для каждой задачи из плана определить **текущее состояние** по GTD:

#### Категории GTD

| Категория | Критерий | Иконка |
|---|---|---|
| **Делай сейчас** | Все данные есть, нет блокеров, можно начать прямо сейчас | **→** |
| **Ожидание** | Мяч на чужой стороне — ждём ответа/данных/решения | **⏳** |
| **Запланировано** | Есть конкретная дата/время (встреча, дедлайн) | **📅** |
| **Заблокировано** | Есть блокер, который нужно сначала устранить | **🚫** |
| **Сделано** | Закрыто с момента последней проверки | **✓** |

#### Определение "следующего физического действия"

Для каждой задачи в категории "Делай сейчас" — сформулировать **конкретное физическое действие**:

- НЕ "Работать над ГТЛК КП"
- А "Открыть шаблон КП, заполнить раздел лицензий на основе данных Акимова (667 сотр., 600 РМ)"

- НЕ "Разобраться с Норникелем"
- А "Прочитать комментарий Храмова в MIMP-891, ответить FYI принято"

#### Правила приоритизации "что делать сейчас"

1. **Кто-то ждёт ответа прямо сейчас** (открытая цепочка из почты) → первый приоритет
2. **Обещал сегодня** (из плана/переписки) → второй приоритет
3. **Лягушка дня** (если не сделана и категория "Делай сейчас") → третий приоритет
4. **Слон дня** (кусочек на сегодня) → четвёртый приоритет
5. **Задачи Q1** с приближающимся дедлайном → пятый приоритет
6. **Задачи Q2-Q4**, которые можно быстро закрыть (<15 мин) → шестой приоритет

## Важные правила

### Философия GTD в навыке
- **"Следующее действие" = физическое действие.** Не "разобраться", а "открыть файл X и написать абзац Y". Не "подумать", а "позвонить Синтикову по номеру +7..."
- **"Ожидание" — тоже результат.** Если по задаче мяч на чужой стороне — это не задача "в работе", это задача "в ожидании". Не нужно ничего делать, нужно дождаться
- **2-минутное правило:** если задачу можно сделать за 2 минуты — она всегда в "Делай сейчас", независимо от квадранта
- **Контекст:** учитывать, что пользователь за компьютером (может писать письма, работать с документами, звонить), но НЕ на встрече и НЕ в дороге

### Архитектура
- **Только чтение.** Навык НЕ пишет в Wiki, НЕ обновляет план, НЕ меняет задачи в Трекере
- **Глубина почты.** Если пришло важное письмо — прочитать содержимое, не ограничиваться заголовком
- **Rate limits.** Максимум 2 параллельных MCP-вызова (почта + календарь). Трекер — последовательно
- **Не дублировать good-morning.** Если good-morning уже был сегодня и новой почты нет — сказать "без изменений, продолжай по плану" и дать GTD-рекомендацию из кэшированного контекста плана

### Тон
- Короткий, действенный
- Не пересказывать весь план — только дельту и рекомендации
- Главный вопрос, на который отвечает навык: **"Что мне делать прямо сейчас и почему именно это?"**
